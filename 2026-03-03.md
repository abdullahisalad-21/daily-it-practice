📘 Overview
Today I troubleshot a failing FTP server on Linux (vsftpd), restored the service, verified FTP connectivity using lftp, 
installed IIS on Windows using PowerShell, and prepared for Windows Server installation to match my tutor’s environment.

🛠️ Tools & Commands Used (with outputs)
lftp initial test
lftp localhost
ls
Interrupt
quit

The FTP control channel connected, but the data channel failed, meaning vsftpd was not listening.
Reloading vsftpd
sudo service vsftpd reload

Reload succeeded silently, but the daemon was still inactive.
Second lftp attempt
lftp localhost
ls
`ls' at 0 [Delaying before reconnect: 27]
Interrupt
quit

The server still wasn’t responding.
Checking service status
sudo systemctl status vsftpd

Output:
Active: failed (Result: exit-code)
Main PID: 5455 (code=exited, status=2)

vsftpd was crashing immediately.
Checking logs
sudo journalctl -u vsftpd -n 50 --no-pager

Output:
status=2/INVALIDARGUMENT

Confirmed a configuration error.
Viewing config
nl -ba /etc/vsftpd.conf

Found the root cause: write_enable=YES was commented out.
Fixing config
Changed:
#write_enable=YES

to:
write_enable=YES

Restarting service
sudo systemctl restart vsftpd

Service started successfully.
Verifying port 21
sudo ss -tulpn | grep :21

Output:
tcp LISTEN 0 32 0.0.0.0:21 users:(("vsftpd",pid=5631,fd=3))

Successful lftp
lftp localhost
ls
Books Desktop Documents ...
quit

Installing IIS
Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebServerRole

Output:
Online        : True
RestartNeeded : False


🚀 Steps Completed
Diagnosed vsftpd crash
Fixed missing write_enable=YES
Restored port 21
Verified FTP functionality
Installed IIS on Windows
Understood difference between client IIS and Windows Server IIS
Prepared for Windows Server 2022 installation

🐞 Problems & Fixes
vsftpd crash → fixed by enabling write_enable=YES
lftp hanging → fixed after restoring port 21
IIS missing server roles → identified OS difference

🧠 What I Learned
Exit code 2 = invalid config
FTP requires both control + data channels
Windows 10/11 IIS ≠ Windows Server IIS
Server Manager roles only exist on Windows Server
