# 🛡️ Module 6 – Security Overview (Daily Practice)

## 1️⃣ Extended Overview
Today I studied core Module 6 security concepts that strengthen an organization’s overall security posture. I explored user‑driven risks, mobile device protections, BYOD threats, vendor risk management, vulnerability scanning, incident response cleanup, and quarantine networks. This session helped me understand how technical controls, policies, and user behavior combine to create a complete security culture.

## 2️⃣ Objective
My goal was to understand how organizations detect, contain, and recover from incidents, how to secure mobile and personal devices, how to manage third‑party risks, and how to enforce strong security practices across users and systems.

## 3️⃣ Tools & Concepts Used
- Vulnerability scanners (Nessus, OpenVAS, Qualys)
- NAC quarantine concepts
- Incident response frameworks
- DRM & EULA concepts
- Regulated data categories (PHI, PCI, PII, FISMA, EAR)
- BYOD, MDM, EMM, MFA
- Mobile OS permission models

## 4️⃣ Steps Completed
- Reviewed user‑security behavior and usability tradeoffs  
- Analyzed mobile security controls (screen locks, encryption, permissions)  
- Studied BYOD risks and mitigation strategies  
- Learned vendor risk assessment and verification processes  
- Reviewed incident cleanup and forensic evidence handling  
- Studied regulated data categories and compliance requirements  
- Understood DRM and EULA enforcement differences  
- Learned NAC quarantine workflows and access control lists  

## 5️⃣ Problems & Fixes
- No technical errors, but the challenge was organizing many security domains.  
- I solved this by grouping topics into: user security, device security, vendor risk, incident response, and compliance.

## 6️⃣ What I Learned
- Users are often the weakest link; security must balance usability.  
- Mobile devices require encryption, permissions, and privacy controls.  
- BYOD introduces major risks without MDM/EMM enforcement.  
- Vendor trust must be verified, not assumed.  
- Incident cleanup requires containment, forensic imaging, and legal coordination.  
- Regulated data must be handled according to strict laws.  
- DRM enforces technical restrictions; EULAs enforce legal restrictions.
