# 🧩 4. Steps Completed  
## 4.1 MFA vs Identification  
- Identification = declaring identity  
- Authentication = proving identity  
- MFA requires multiple factor categories  

## 4.2 HOTP vs TOTP  
- HOTP = counter‑based, no expiration  
- TOTP = time‑based, expires every 30 seconds  
- TOTP is more secure and industry standard  

## 4.3 Biometrics  
- Fingerprint: convenient but spoofable  
- Facial recognition: 2D weak, 3D + IR stronger  
- Iris: vulnerable to photo spoofing  
- Retina: strongest but expensive  
- Biometrics require enrollment + authentication  

## 4.4 Physical Authentication Factors  
- NFC: short‑range, combines “have” + “are”  
- GPS: location‑based authentication  
- IPS: indoor triangulation  
- Geofencing: radius‑based access control  

## 4.5 RADIUS Architecture  
- NAS → RADIUS Proxy → RADIUS Server  
- Centralized AAA  
- Used for Wi‑Fi, VPN, ISP authentication  

## 4.6 TACACS+ vs XTACACS vs RADIUS  
- XTACACS: obsolete  
- TACACS+: full packet encryption, device admin  
- RADIUS: network access, password‑only encryption  

## 4.7 Kerberos  
- AS_REQ → AS_REP (TGT)  
- TGS_REQ → TGS_REP (Service Ticket)  
- AP_REQ → AP_REP (Service Access)  
- KDC = AS + TGS  

## 4.8 SSO  
- One login → multiple services  
- Kerberos = Windows SSO  
- OIDC = modern web SSO  
- SAML = enterprise SSO  
- OAuth2 = authorization; OIDC adds authentication

# 📚 6. What I Learned  
- AAA is the foundation of enterprise security  
- MFA must combine different factor categories  
- Biometrics can be spoofed; require liveness detection  
- RADIUS and TACACS+ serve different AAA roles  
- Kerberos avoids sending passwords over the network  
- OIDC is the modern standard for web SSO  
- Physical factors add strong contextual security  
- SSO improves security but must be protected with MFA
